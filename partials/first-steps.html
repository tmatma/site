First Steps

Kickstrap is a boilerplate of Bootstrap, JSPM, Angular.js and Roots that can copy itself into any directory you choose.

If you're looking at this page from your local installation, you've successfully got Kickstrap up and running with the example pages. You should skip to Editing content. Otherwise, continue reading.

Need to ask a question?

Create an issue on the Kickstrap GitHub page

1. Install Kickstrap

To install Kickstrap, you'll need Node Package Manager. . If you already have this installed, run

sudo npm update -g

Then...

sudo npm install -g kickstrap

2. Create a new Kickstrap site

Navigate to the directory where you'd like to create a new Kickstrap site and run 

kickstrap new mysite

This will create a new directory called "mysite" with the Kickstrap boilerplate. If instead you want to make the site in current directory, replace "mysite" with "." 

kickstrap new .

When the files have finished transferring, Roots will automatically start a local server in that directory and reload when any changes are detected (Live Reload).

To stop this process, press ctrl + c. To run the watcher again run either 

kickstrap watch

or 

roots watch

If something isn't right, file or search your issue at the Kickstrap GitHub page.
If the issue is related to Roots compilation make sure you have satisfied the requirements here.

Continue reading: Compiling in Roots to learn how best to compile, watch, and prepare for a production website.

3. Create your own Firebase account

In its current form, your Kickstrap example pages are hooked up to a read-only Firebase. This will prevent you from doing anything truly useful in a production environment since users can't manipulate data.

Fortunately, Firebase offers free accounts under a certain usage tier. You can sign up for a Firebase account here: firebase.com/signup

Once you have your first Firebase, just open js/_settings.coffee... 

# Firebase URI
# firebaseName: 'kickstrap-demo'

Uncomment firebaseName, and replace it with your own url. This will be the name before ".firebaseio.com" For example, https://mysite.firebaseio.com would be 

# Firebase URI
firebaseName: 'mysite'

Now you have your own Firebase, but you'll notice all the data is empty. If you want to reinstall the sample data, I've included a .json file you can export into firebase. Look for kickstrap-demo-data.json in the root of your project. Import this into Firebase by visiting https://[yoursitename].firebaseio.com, clicking on the data tab, then clicking "Import JSON".

Please note, this will delete any data you have there.

4. Editing content

tl;dr: The easiest way to make a page is to create a new .jade file in /views

Also see:

  • Creating pages

In Roots, your pages are generated with the files in views/.

layout.jade and blank.jade are base files. By default, files will extend (add themselves to) blank.jade. This is the case for partials and layout.jade, each of which we'll explain next.

The pages in the example app extend layout.jade, which provides the repetitive header and footer content you'll use application-wide. In the case of this page, partials are used inside index.jade via angular. This allows a single page to refresh with content for only a small part of the page. By loading only the partial, all the head content and JavaScript dependencies do not need to be set up again.

Kickstrap uses both Angular partials and Jade partials. Angular partials load in real time, on request in the browser. They can be found in views/partials/.

Jade partials are compiled once in the project and added to the final .html files. These are found in views/ and should start with an underscore. (e.g. "_footer.jade")

Change theme and basic settings

Import one of the included Bootstrap themes into main.css. These are found in /themes

To do this, open main.css and change the theme imported at the top

@import "../theme/[yourthemenamehere].css"

Editing the header

The header that provides the navigation bar is a Jade partial. This is automatically added to any view with the code: 

include _header
_header.jade can be found in views/.
