Intro to Stylus

Stylus is a serious time-saver for writing CSS

This will not be an exhaustive document covering Stylus but mainly covering conventions you'll see used in Kickstrap. For complete documentation, visit the Stylus homepage.

The beauty of Stylus is its ability to read plain CSS. You don't technically have to learn any new syntax to jump into Stylus. For example:

a {
  color: #428BCA;
}
a.active {
  color: white;
  background-color: #428BCA;
}
a.active span {
  background-color: yellow;
}

...Is totally valid stylus. However, stylus lets us get rid of those curly braces and semicolons if we indicate nesting by tabbing in.

a
  color: #428BCA
a.active
  color: white
  background-color: #428BCA
a.active span
  background-color: yellow

We can also nest on property under another like this:

a
  color: #428BCA
  &.active
    color: white
    background-color: #428BCA
    span
      background-color: yellow

The ampersands (&) denote concatenation. The absence of an ampersand will put a space between identifiers.

We can also use variables in Stylus and get rid of that repetitive hex for blue.

$blue = #428BCA
a
  color: $blue
  &.active
    color: white
    background-color: $blue
    span
      background-color: yellow
